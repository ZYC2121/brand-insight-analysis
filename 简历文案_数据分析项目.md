## 推荐版本（简历主体用）

### Brand Insight AI — 消费者数据分析工作台 | 独立开发项目（2025.07）

*【分析框架设计 → AI Agent驱动自动化 → 可视化报告输出，构建可复用的人×AI协作分析工作流】*

> - **通用分析引擎开发**：基于 Python + Streamlit 构建**数据-类型自动识别分析引擎**，可自动识别任意 CSV 数据集的列类型（数值/分类/日期/ID），并根据数据特征自动匹配分析方法（描述统计、相关性分析、t检验/ANOVA、K-Means聚类），实现"上传任意数据 → 自动生成分析报告"的完整闭环。
> - **AI Agent 驱动全流程**：运用 Claude Agent 完成核心引擎代码生成、统计分析算法实现与可视化模块开发，实践 **"人定义分析逻辑、AI执行代码实现"** 的新型开发范式，将传统需多人协作的分析工具开发周期从数周压缩至半天。
> - **多数据验证与策略输出**：使用 Kaggle 消费者购物数据集（3,900条）和云营销策略数据集（5,000条）进行双数据验证，通过统计检验识别出季节因素对消费金额的显著影响（p=0.011），输出基于数据信号的分群营销策略建议。
> - **可复用工作流沉淀**：项目封装为通用 CLI + Web 双模式工具（`python auto_analyzer.py <csv_path>` 或 `streamlit run app.py`），任一营销数据集拖入即可获得完整分析报告，实现了分析能力的工具化沉淀与复用。

*关键词命中：数据分析、消费者洞察、AI工具应用、自动化工作流、策略输出、Python/Streamlit*

## 面试 STAR 问答

### S - Situation
在做消费者数据分析项目的过程中意识到：每次数据分析都要从头写脚本，效率低且不可复用。如果能构建一个通用的自动分析引擎，任何营销数据拖入就能出报告，不仅自己能用，团队也能用，会是更有价值的产出。

### T - Task
目标是将单一的消费者数据分析项目升级为通用的数据分析工作台：用户上传任意 CSV 数据 → 引擎自动识别数据特征 → 匹配分析方法 → 输出可视化图表和分析报告。

### A - Action
- 设计了通用分析架构：列类型自动检测（数值/分类/日期/ID）→ 方法自动匹配 → 报告自动生成
- 使用 Claude Agent 生成核心引擎代码（~400行 Python），实现了描述统计、相关性矩阵、t检验/ANOVA、K-Means聚类 等模块
- 构建了 Streamlit 网页交互界面，支持文件拖拽上传、实时进度展示、可视化结果浏览和报告下载
- 用两个不同结构的数据集（3,900条消费者数据 + 5,000条营销策略数据）进行了验证，引擎均能正确识别和分析
- 封装为 CLI + Web 双模式：命令行一句话出报告、网页拖拽式交互，两种使用方式

### R - Result
- 开发了一个通用的、可复用的消费者数据分析工作台
- 两个数据集验证通过，引擎具备真正的通用性
- Web 界面可随时演示给面试官看
- 该项目直接体现了：数据分析方法论 × AI工具驾驭能力 × 产品化思维 × 从0到1的构建能力


## 演示方式

### 命令行模式
```bash
# 分析任意 CSV 数据，一句话出报告
python auto_analyzer.py data/your_file.csv output/your_analysis
```

### Web 界面模式
```bash
# 启动网页，拖拽上传 CSV 即可
streamlit run app.py
```

### 输出内容
- 描述统计表格
- 单变量分布图（直方图+箱线图+分类柱状图）
- 相关性热力图
- t检验/ANOVA 显著性检验结果
- K-Means 客户聚类 + PCA 可视化
- 完整 Markdown 分析报告
