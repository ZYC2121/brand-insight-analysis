## 六项升级明细

### 1. 数据质量检测（新增）

**位置**: `auto_analyzer.py` → `data_quality()` 方法（46 行）

**之前**: 代码只统计缺失值数量，不做任何级别判断。异常值完全未检测。

**之后**:
- **缺失值**: 按缺失率分三级——<5% 低风险、5%-20% 中风险、>20% 高风险
- **异常值**: IQR 方法（Q1 - 1.5×IQR / Q3 + 1.5×IQR），同样分三级
- 结果写入 `meta['data_quality']`，报告中自动展示警告表格
- Overview 标签页弹出 `st.warning` 提醒用户

**面试怎么讲**: "数据分析师 60% 的时间花在数据清洗上。我做了自动化检测——缺失率超过 20% 的列会标红警告，异常值用 IQR 筛选。分析结论的可靠性前提是数据质量。"

---

### 2. Pearson + Spearman 双方法相关性分析（升级）

**位置**: `auto_analyzer.py` → `correlation_analysis()` 方法（重写）

**之前**: 只计算 Pearson 线性相关系数。

**之后**:
- 同时计算 Pearson 和 Spearman 矩阵，生成双热力图并列展示
- 高相关对表格同时显示两种方法的 r 值
- **新增**: 自动检测 Pearson 与 Spearman 差异 >0.2 的变量对，标记为"可能存在非线性单调关系"提示
- 报告分为"显著相关"和"非线性关系提示"两部分

**面试怎么讲**: "Pearson 只能抓线性关系。如果两个变量是幂律关系，Pearson 接近 0 但 Spearman 可能 >0.8。我把两种方法并列对比，差异大时自动提醒用户——这本身就是一个分析洞察。"

---

### 3. K-Means 聚类增加轮廓系数（升级）

**位置**: `auto_analyzer.py` → `clustering()` 方法（重写）

**之前**: 只用肘部法则（Inertia 二阶差分）选 K 值，K 硬限制在 2-5。

**之后**:
- 同时计算肘部法则（Inertia）和轮廓系数（Silhouette Score，范围 [-1, 1]）
- K 值选择策略：轮廓系数优先，但肘部 K 更大且轮廓系数差距 <0.05 时取更大的
- K 上限提高到 6
- 聚类结果附轮廓系数质量评级——>0.5 良好、>0.25 可接受、<0.25 较差
- 图表改为双 Y 轴（Inertia + Silhouette 同图）

**面试怎么讲**: "肘部法则太主观——你看到的'肘部'别人可能觉得是直线。我加了轮廓系数做交叉验证，两个指标一起决定 K 值。聚类质量好坏有客观数字而不是凭感觉。"

---

### 4. 效应量标准化为 Cohen's d / η² / r（升级）

**位置**: `auto_analyzer.py` → `group_comparison()` 方法

**之前**: 使用自创的 (max_mean - min_mean) / overall_std 公式，非标准，面试无法解释。

**之后**:

| 检验方法 | 效应量指标 | 解读标准 |
|---|---|---|
| t-test | **Cohen's d** | >0.8 大、>0.5 中、>0.2 小 |
| ANOVA | **η² (eta-squared)** | >0.14 大、>0.06 中、>0.01 小 |
| Kruskal-Wallis | **η²_H** | 同上 |
| Mann-Whitney U | **r (rank-biserial)** | >0.5 大、>0.3 中、>0.1 小 |

报告中每个显著结果都标注效应量名称、数值和大小解读。

**面试怎么讲**: "p 值只能告诉你'有没有差异'，效应量告诉你'差异有多大'。比如 p=0.001 但 Cohen's d=0.1，统计显著但实际意义很小——不看效应量容易被 p 值骗。"

---

### 5. 自动策略建议生成（新增）

**位置**: `report_generator.py` → 新增第八章"策略建议"

**之前**: 报告以"关键发现总结"结尾，纯描述性。

**之后**: 新章节基于四项分析结果自动生成可操作建议：

| 数据来源 | 生成逻辑 |
|---|---|
| **驱动力分析** | 模型显著 → 推荐聚焦 Top 驱动因素配置资源 |
| **分组对比** | Bonferroni 显著 + 事后检验 → 推荐差异化干预策略 |
| **客户聚类** | 轮廓系数 >0.25 → 推荐分群运营策略 |
| **相关性非线性提示** | Pearson vs Spearman 差异大 → 建议深入探索非线性关系 |

**面试怎么讲**: "分析的目的不是出报告，是指导决策。我在每个分析模块后面加了一层'所以呢'——Top 3 驱动因素是什么、哪个客群值得投资源、差异最大的是哪两组。从描述到诊断到建议，形成闭环。"

## 当前能力定位

```
  指导性 (Prescriptive) → "我们应该做什么？"  【v2.1 新增: ✅ 策略建议】
  诊断性 (Diagnostic)   → "为什么会发生？"    【✅ 驱动力分析 + 分组检验 + 事后检验】
  描述性 (Descriptive)  → "发生了什么？"      【✅ 完整覆盖】
  预测性 (Predictive)   → "接下来会发生什么？" 【✗ 下一阶段】
```

本次升级使平台从描述+诊断层向指导层迈出了关键一步。
